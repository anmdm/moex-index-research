# Синтез оптимального актива для mean-reversion стратегии на MOEX

## Ключевая идея

Вместо подбора сложной стратегии под существующий инструмент — **конструируем идеальный синтетический актив под простейшую стратегию** (grid/сетку).

На рынке существуют две фундаментальные стратегии: **флетовая** (mean-reversion) и **трендовая** (momentum). Каждая из них — функция-фильтр, которая извлекает определённый тип движения из ценового ряда. Если стратегий всего две, зачем строить сложные комбинаторные системы с 15-25 параметрами? Нужно найти актив, который *идеально* подходит под одну из двух базовых стратегий, и тогда стратегия сводится к простейшей форме.

**Результат:** синтетический индекс из 3 товарных фьючерсов MOEX (нефть Brent, природный газ, серебро) с grid-стратегией, имеющей всего 5 параметров, показывает **+259% за 3 года при MaxDD -12.5% (P&L/DD = 20.7)** на тиковых данных с реальными комиссиями и задержками исполнения.

## Концепция

### Классический подход и его проблемы

Традиционный количественный трейдинг: взять один инструмент и подобрать стратегию. Это приводит к:

- **Комбинаторному взрыву параметров** — фильтры режимов, адаптивные пороги, множество условий входа/выхода (10-25 параметров)
- **Переобучению** — чем больше параметров, тем лучше fit на исторических данных и тем хуже результат в реальном времени
- **Хрупкости** — любое изменение рыночного режима ломает стратегию

### Инвертированный подход

**Зачем подбирать стратегию под актив, если можно подобрать актив под стратегию?**

На бирже торгуются сотни инструментов. Из них можно сконструировать **синтетический актив** (взвешенную корзину), обладающий нужными статистическими свойствами. Для mean-reversion стратегии нужен актив, который:

1. **Максимально осциллирует вокруг среднего** — высокая oscillation volatility
2. **Быстро возвращается к среднему** — короткий half-life (но не быстрее ~2 мин, чтобы можно было отторговать)
3. **Минимально дрифтует** — drift << oscillation, чтобы средняя не уезжала
4. **Стабилен по годам** — oscillation/trend ratio стабилен на разных временных масштабах (микро и макро)
5. **Long bias при наличии тренда** — если дрифт неизбежен, пусть будет вверх (long-only позиции)

Когда актив идеально подходит, стратегия сводится к **простейшей сетке (grid)**: вход при отклонении от среднего, выход при возврате. 5 параметров вместо 25.

### Почему конструкция устойчива

**Три источника дохода:**

| Режим рынка | Источник дохода |
|------------|----------------|
| Флет | Сетка собирает волатильность (oscillation) |
| Тренд вверх | Long-only позиции едут с трендом + девальвация рубля |
| Шок одного актива | Раскорреляция (нефть ↓ → золото ↑) удерживает индекс, сетка работает |

**Единственный опасный сценарий:** одновременное падение ВСЕХ commodities в долларах + существенное укрепление рубля. В текущей геополитике (санкции, бюджетный дефицит, военные расходы) — крайне маловероятно.

|-------|-----------|-------|--------|
| BR | Фьючерс на Brent | 24.1M | 184 MB |

| GD | Фьючерс на золото | 19.5M | 154 MB |
| SV | Фьючерс на серебро | 17.8M | 135 MB |
| MX | Фьючерс на индекс МосБиржи | 17.9M | 138 MB |

### Обработка
1. Дедупликация partial fills (одинаковый timestamp + direction = один ордер, разбитый на части)
2. Resample до 5-секундных баров (last price)
3. Outer join + forward-fill (когда тик приходит по одному инструменту, остальные берут последнюю известную цену)
4. Результат: **18.9M** синхронизированных 5-секундных баров

### Контракты
Фьючерсы имеют срок экспирации. Для каждого инструмента загружаются все фронт-контракты за период, автоматически определяется наиболее ликвидный контракт на каждую дату. При ролловке (переход с одного контракта на другой) возникает ценовой скачок — это реальность, не артефакт.

## Построение индекса

### Формула

```
Index(t) = Σ(w_i × price_i(t) / SMA_i(t))
```

Где:
- `w_i` — вес i-го инструмента (нормализованы, Σw_i = 1)
- `price_i(t)` — текущая цена i-го инструмента
- `SMA_i(t)` — скользящая средняя цены с окном 600 секунд (120 баров × 5с)

Деление на SMA нормализует каждый актив к его "среднему уровню", превращая абсолютные цены в относительные отклонения.

Z-score показывает отклонение индекса от нормы:

```
Z(t) = (log(Index(t)) - mean(log(Index), lookback)) / std(log(Index), lookback)
```

Lookback = 7865 секунд (~2.2 часа, 1573 бара × 5с).

### Multi-scale скоринг индекса

Ключевая метрика при поиске оптимальных весов — **oscillation/trend ratio** на нескольких масштабах:

1. **Micro-scale (10-сек бары):** oscillation vol = std отклонений от SMA
2. **Macro-scale (дневные бары):** drift = abs(годовой тренд log(Index))
3. **Yearly uniformity:** CV (coefficient of variation) среднего уровня индекса по годам

Score = `oscillation_vol × (100 / half_life) × drift_penalty × yearly_uniformity`

Drift penalty штрафует **только нисходящий** тренд. Восходящий тренд допускается (long-only позиции).

### Найденный индекс

Optuna (2000 trials, TPE sampler) нашла оптимальные веса:

| Инструмент | Вес | Роль в индексе |
|-----------|-----|---------------|
| **BR (Brent)** | **36%** | Энергоноситель, высокая волатильность, раскоррелирован с металлами |
| **NG (Газ)** | **28%** | Энергоноситель, сезонные и политические факторы, слабо связан с нефтью |
| **SV (Серебро)** | **36%** | Промышленный + защитный металл, контрциклический |

**Исключённые инструменты:**
- GD (золото) — меньше добавляет oscillation чем серебро
- MX (МосБиржа) — трендовый на макро-масштабе
- Si, CR (валюты) — однонаправленный тренд (девальвация), убивает mean-reversion

## Стратегия

### Variant D: Group-TP Grid

Лучший из 5 протестированных вариантов — **Group-TP**: все позиции группы закрываются одновременно, когда средневзвешенный портфельный P&L достигает порога.

#### Параметры (5 штук)

| Параметр | Значение | Описание |
|----------|---------|----------|
| `n_parts` | 4 | Капитал делится на 4 части |
| `entry_z` | 2.44 | Вход при Z-score < -2.44 |
| `grid_step_z` | 0.60 | Следующий вход через 0.6σ ниже |
| `tp_pct` | 0.36% | TP при средневзвешенном portfolio return ≥ 0.36% |
| `max_pos` | 4 | Максимум 4 одновременных позиции |

#### Логика

1. Ждём пока Z-score индекса опустится ниже -2.44 (индекс ниже среднего на 2.44σ)
2. Покупаем корзину: 36% в BR, 28% в NG, 36% в SV (каждый инструмент отдельно)
3. Если Z продолжает падать — добавляем позицию каждые 0.6σ (до 4 позиций)
4. Считаем средневзвешенный P&L ВСЕХ открытых позиций: `Σ(w_k × (price_now_k / price_entry_k - 1))`
5. Когда avg P&L ≥ 0.36% — закрываем ВСЕ позиции сразу
6. Комиссия (round-trip) вычитается из P&L каждой позиции

#### Почему Group-TP лучше индивидуального TP

В Group-TP убыточные позиции "вытягиваются" прибыльными. Если первая позиция в минусе, но вторая (вошли ниже) в большом плюсе — суммарно группа в прибыли и закрывается. Это позволяет:
- Более агрессивные входы (entry_z = 2.44 vs 4.0 для Base)
- Более частые сделки (859 vs 2521)
- Лучшую равномерность по годам

#### Бэктестер: ключевые свойства

- **Cash management:** позиция не открывается если free_cash < pos_size
- **Execution delay:** 1 бар (5 секунд) между сигналом и исполнением
- **Session filter:** торговля только в сессии MOEX (Main 10:00-18:45 MSK, Evening 19:05-23:50 MSK)
- **Торговля инструментами, не индексом:** каждый инструмент покупается/продаётся отдельно
- **Комиссии:** 0.025% round-trip на каждую сделку

## Сравнение с одиночными активами

Та же стратегия D (Group-TP), оптимизированная отдельно для каждого инструмента:

| Актив | P&L | MaxDD | P&L/DD | 2023 | 2024 | 2025 | 2026 |
|-------|-----|-------|--------|------|------|------|------|
| **Index (36%BR+28%NG+36%SV)** | **+259%** | **-12.5%** | **20.7** | **+24%** | **+20%** | +124% | **+9%** |
| SV solo (серебро) | +301% | -16.0% | 18.8 | +8% | +13% | +228% | +0% |
| NG solo (газ) | +69% | -16.4% | 4.2 | +6% | +33% | +12% | +7% |
| BR solo (нефть) | +55% | -12.5% | 4.4 | +1% | +0% | +35% | +12% |

### Выводы

1. **Индекс P&L/DD=20.7 — в 5× лучше чем NG (4.2) и BR (4.4)**, при одинаковом DD
2. **SV solo** зарабатывает больше абсолютно (+301%), но почти всё на тренде 2025 (+228%). В 2023-2024 SV даёт +8/+13% — индекс +24/+20%
3. **Раскорреляция работает**: NG лучший в 2024 (+33%), SV в 2025 (+228%), BR в 2026 (+12%). Индекс собирает лучшее от каждого
4. **DD**: индекс и BR одинаковый DD (-12.5%), но индекс даёт +259% vs BR +55%. Диверсификация не снижает DD, но драматически увеличивает P&L

**Идея подтверждена: синтетический индекс из 3 товаров превосходит каждый актив по отдельности по P&L/DD и стабильности.**

2. **TMON — безрисковый бенчмарк** (DD=-0.3%, Sharpe=9.8). Это по сути ставка ЦБ РФ (~17-18% годовых). По risk-adjusted метрикам его невозможно побить, но это пассивный доход без upside.

5. **Sortino INDEX (3.91) > TRUR (3.16)** — при учёте только downside risk (что важнее для инвестора) INDEX эффективнее.
6. **INDEX берёт не за счёт низкого риска, а за счёт экстремальной доходности при контролируемом DD.** MaxDD -12.5% — это приемлемо для активной стратегии.

## Варианты стратегий

Протестировано 5 вариантов grid-стратегии на одном индексе:

| Вариант | Идея | P&L | DD | P&L/DD |
|---------|------|-----|-----|--------|
| **A: Base** | Фиксированный TP для каждой позиции | +33% | -7% | 4.9 |
| **B: Dev-TP** | TP масштабируется глубиной входа (больше отклонение → больше TP) | +3% | -23% | 0.1 |
| **C: Progressive** | Размер позиции + TP масштабируются глубиной | +752% | -4720% | 0.2 |
| **D: Group-TP** | Все позиции закрываются вместе по средневзвешенному P&L | **+259%** | **-12.5%** | **20.7** |
| **E: Vol-adaptive** | Шаг сетки и TP масштабируются текущей волатильностью | +38% | -10% | 3.8 |

**D (Group-TP)** — явный победитель по всем метрикам.

> Примечание: B и C показали плохие результаты потому что adaptive TP при больших отклонениях заставляет держать позиции слишком долго, а progressive sizing без ограничений уходит в margin call.

## Путь исследования

### Хронология (рабочие и нерабочие идеи)

**Этап 1: Выбор площадки и инструментов**
- 7 фьючерсов: BR, NG, GD, SV, MX, Si, CR
- Минутные данные → переход на тиковые для реалистичности
- Первый индекс: 22%BR+15%NG+25%GD+27%SV+11%MX (старая методология)

**Этап 3: Поиск индекса (нерабочие попытки)**
- v1: Max DD scoring → нашёл 4%BR+23%NG+15%SV+58%MX, P&L/DD=6.6
- v2: Heavy DD penalty → 11%SV+89%MX (2 актива, слишком простой)
- v3: Max volatility scoring → 5%NG+79%SV+16%MX (+316%, но 79% серебро = тренд, не mean-reversion, годы [+16/+22/+120/+7])
- **Проблема v3**: osc/trend ratio считался на 10-сек барах. На микро-масштабе серебро осциллирует, на макро — трендит

**Этап 4: Multi-scale scoring (рабочая версия)**
- Oscillation/trend ratio на ДВУХ масштабах (10-сек + дневной)
- Yearly uniformity penalty
- Max weight per asset: 35% (запрет моно-индекса)
- Результат: **36%BR+28%NG+36%SV** — сбалансированная товарная корзина

**Этап 5: Стратегия**
- 5 вариантов (A-E) тестировались параллельно
- D (Group-TP) — победитель
- Cash management добавлен после обнаружения бага (позиции открывались при нулевом кэше)

**Этап 6: Валидация**
- NautilusTrader: +96% (conservative estimate, меньше trades из-за 30-сек баров)
- VectorBT: +77% (наиболее консервативный)
- Fast BT с cash management: +259% (основной результат)
- Сравнение с solo-инструментами: индекс в 5× лучше по P&L/DD

### Нерабочие идеи (проверенные и отвергнутые)
- **Martingale sizing** — DD слишком высокий
- **Stop-loss (time-based)** — ухудшает P&L/DD
- **Vol_tp_mult / dev_tp_mult / trail_pct** — нет эффекта (close-all доминирует)
- **Adaptive Grid** — DD растёт пропорционально P&L
- **Валюты в индексе (Si, CR)** — трендовые, убивают mean-reversion
- **Золото (GD)** — исключено multi-scale скорингом (менее эффективно чем серебро)
- **Micro-TP (0.1%)** — убивается execution delay

## Запуск

### Установка зависимостей

```bash
pip install requests pandas numpy plotly statsmodels optuna pyarrow numba
```

### 1. Загрузка данных (требуется T-Invest API токен)

```bash
python src/moex_download_robust.py
python src/moex_convert_ticks.py
```

### 2. Поиск индекса + оптимизация стратегии

```bash
python src/moex_pipeline_v5.py
```

### 3. Оптимизация стратегии D на найденном индексе

```bash
python src/moex_optimize_d.py
```

### 4. Сравнение с одиночными активами

```bash
python src/moex_optimize_sv_solo.py
python src/moex_optimize_ng_solo.py
python src/moex_optimize_br_solo.py
```

## Философия

> Не ищи сложную стратегию для случайного актива.
> Собери идеальный актив для простой стратегии.

Простая стратегия на правильном активе бьёт сложную стратегию на случайном — потому что у неё меньше параметров, меньше переобучения, понятный edge и прозрачная логика отказа.
